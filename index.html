3.1.1 · Obj 2🔗 Types of Networks — Identify, Define, Compare
▼
📋 AS 3.1.1: Identify different types of networks. Examples include: LAN, VLAN, WAN, SAN, WLAN, internet, extranet, VPN, PAN, P2P.
S/E, INT: Globalization has been accelerated by the technical advances linked to network development.

Core Network Types — Define, Compare, Example

LAN — Local Area Network
Definition: A network that connects devices within a small geographical area (single building or site) using private infrastructure.
Adv: High bandwidth, fast speeds, low cost, easy to manage, more secure.
Disadv: Limited range; expensive to extend over large distances.
e.g. School computer lab, office building network
WAN — Wide Area Network
Definition: A network covering a large geographical area (national/global), typically using public infrastructure (e.g. telephone lines).
Adv: Connects geographically distant sites; enables global communication.
Disadv: Slower, less secure, more expensive to maintain; depends on third-party providers.
e.g. The internet, bank connecting branches nationwide
WLAN — Wireless LAN
Definition: A LAN that uses wireless connections (radio waves) instead of physical cables.
Adv: Mobility, easy to expand, no cable installation needed.
Disadv: Security risks, interference, generally slower than wired.
e.g. School Wi-Fi network, home wireless router
VLAN — Virtual LAN
Definition: A logical subdivision of a physical network — devices are grouped into a virtual network regardless of their physical location, configured via software (switches).
Adv: Improved security (isolates groups), reduces broadcast traffic, flexible.
Disadv: Requires managed switches; complex configuration.
e.g. Finance dept isolated from HR on same physical network
VPN — Virtual Private Network
Definition: A secure, encrypted connection (tunnel) created over a public network (e.g. the internet) that allows remote users to access a private network as if they were physically connected to it.
Adv: Secure remote access, data encrypted, bypasses geographic restrictions.
Disadv: Slower due to encryption overhead, requires hardware/software investment.
e.g. Teacher accessing school network from home
PAN — Personal Area Network
Definition: A very short-range network (typically ≈10 metres) connecting personal devices around an individual. Uses Bluetooth, infrared, or NFC.
Adv: Very low cost, convenient, cable-free.
Disadv: Very limited range; susceptible to interference.
e.g. Smartphone → wireless earbuds, smartwatch → phone
SAN — Storage Area Network
Definition: A dedicated high-speed network that connects servers to storage devices (e.g. disk arrays), making storage appear as locally attached to each server.
Adv: Very fast data access, centralises storage management, highly scalable.
Disadv: Very expensive to install and maintain; specialist knowledge required.
e.g. Large company data centres, cloud storage infrastructure
MAN — Metropolitan Area Network
Definition: A network that spans a city or large campus — larger than a LAN but smaller than a WAN. Often uses fibre optic.
Adv: Faster than WAN; connects multiple buildings in a city.
Disadv: High setup cost; complex to manage.
e.g. City-wide Wi-Fi, university multi-campus network
P2P — Peer-to-Peer
Definition: A network architecture where every device (node) acts as both client AND server — there is no central server. Nodes share resources directly with each other.
Adv: No single point of failure, no server cost, scales easily.
Disadv: Difficult to manage/secure, performance depends on all peers being online.
e.g. BitTorrent, blockchain networks

Key IB Exam Differences: LAN vs WAN

1
WAN covers a much larger geographical area (national/international); LAN covers a small area (single site/building)
2
WAN nodes use connections through public networks (e.g. telephone system); LAN uses private infrastructure
3
LANs are more secure than WANs (data travels less distance, over private lines)
4
LANs have higher bandwidth and faster data transfer rates than WANs
5

6

3.1.2–3.1.3 · Obj 2📶 Standards & the OSI 7-Layer Model
▼
📋 AS 3.1.2: Outline the importance of standards in the construction of networks. INT: Standards enable compatibility through a common "language" internationally.
AS 3.1.3: Describe how communication over networks is broken down into different layers. Awareness of the OSI 7-layer model is required, but understanding of the functioning of each layer is NOT required.

Why Standards Matter (3.1.2)

NETWORK STANDARD A formally agreed set of technical specifications that define how devices communicate. Standards ensure that hardware and software from different manufacturers can interoperate.
1
Interoperability: A phone made in South Korea can connect to a router made in Germany because both follow the same standards (e.g. IEEE 802.11 for Wi-Fi)
2
International compatibility (INT): Standards provide a common "language" allowing global communication and trade
3
Competition: Multiple manufacturers can build compatible products, driving innovation and lowering prices
4
Reliability: Standardised protocols ensure consistent, predictable network behaviour
📌 Examples of standards: IEEE 802.11 (Wi-Fi) · IEEE 802.3 (Ethernet) · TCP/IP · HTTP/HTTPS · USB · Bluetooth

The OSI 7-Layer Model (3.1.3)

The Open Systems Interconnection (OSI) model describes how data communication is divided into 7 standardised layers. Each layer has specific responsibilities and communicates with the layers directly above and below it. This layered approach means changes in one layer don't affect others.

Mnemonic (Layer 7 → Layer 1): All People Seem To Need Data Processing

7
Application
Provides the interface between end-user applications and the network. Handles HTTP requests, email, file transfers. Examples: HTTP, FTP, SMTP, DNS, SSH
6
Presentation
Translates data formats between the application and network. Handles encryption/decryption, compression, and character encoding. Examples: SSL/TLS encryption, JPEG, ASCII, Unicode
5
Session
Manages sessions (connections) between applications — establishes, maintains and terminates communication sessions. Examples: NetBIOS, SQL sessions, RPC
4
Transport
Provides end-to-end data delivery, error checking, and flow control. Segments data into chunks for network layer. Examples: TCP (reliable) · UDP (fast, no guarantee)
3
Network
Routes packets across different networks using logical addressing (IP addresses). Determines best path for data. Examples: IP, ICMP, routers operate here
2
Data Link
Node-to-node data transfer on the same network using physical (MAC) addresses. Error detection at this level. Examples: Ethernet (802.3), Wi-Fi (802.11), switches operate here
1
Physical
Transmits raw binary bits over the physical medium (cables, radio waves). Defines voltages, timing, pin layouts. Examples: Ethernet cable, fibre optic, radio waves, hubs operate here
✏️ IB Exam Note: You only need awareness of the 7-layer model — know the layer names, numbers, and basic purpose. You are NOT required to explain the detailed functioning of each layer. Focus on knowing the mnemonic and what happens at layers 1, 2, 3, 4, and 7 — these come up most.

Why Use Layers? The Benefits

▸
Modularity: Each layer can be developed and updated independently without affecting others
▸
Standardisation: Different manufacturers can make compatible hardware/software if they follow the same layer specifications
▸
Troubleshooting: Easier to isolate problems — if Layer 3 (routing) is at fault, Layers 1–2 may still work fine
▸
Abstraction: Higher layers don't need to know the details of how lower layers work

▼
📋 AS 3.1.4: Identify the technologies required to provide a VPN. (Obj 2)
AS 3.1.5: Evaluate the use of a VPN. S/E, AIM 9: The use of a VPN has led to changes in working patterns. (Obj 3 — must give both advantages AND disadvantages, with justification)

What is a VPN?

VPN — VIRTUAL PRIVATE NETWORK A VPN extends a private network across a public network (usually the internet), allowing remote users to send and receive data as if their devices were directly connected to the private network. It achieves this through tunnelling (encapsulation of data packets) and encryption.

Technologies Required to Provide a VPN (3.1.4)

1
Client VPN software — installed on the end user's device. Creates the secure tunnel and encrypts outgoing data. Without this, the user's device cannot connect to the VPN.
2
VPN-aware routers and firewalls — configured to permit VPN traffic to pass through. Standard firewalls would block encrypted VPN tunnels without this configuration.
3
VPN appliance/server — server-side hardware/software that handles incoming VPN connections, manages authentication, and routes traffic to the internal network.
4
Encryption protocols — IPSec or SSL/TLS — the mathematical algorithms used to encrypt data in the tunnel. IPSec works at Layer 3 (Network); SSL/TLS works at Layer 4+ (Transport/Application).

How a VPN Works — Step by Step

1
User's device runs VPN client software and requests a connection to the VPN server
2
Authentication: The user proves identity using credentials (username/password, certificates, one-time passwords (OTPs), or software tokens)
3
Tunnel created: A secure encrypted tunnel is established between the client device and the VPN server over the public internet
4
Encapsulation: Each data packet is encrypted and "wrapped" inside another packet (tunnelling) — the original packet becomes the payload of a new one
5
Transmission: Encrypted packets travel across the public internet — even if intercepted, the data is unreadable without the decryption key
6
At the VPN server: Packets are decrypted and forwarded to the destination on the private network
7
The user's device takes on the IP address of the VPN server, masking their real location

Security Features of a VPN

FeatureHow it Provides Security
EncryptionEach encapsulated data packet is encrypted with a key shared only between VPN server and client. Even if intercepted, data cannot be read without the key.
TunnellingData is encapsulated inside another packet, hiding the original packet's contents and routing information from anyone monitoring the network.
AuthenticationStrong multi-factor authentication (certificates, OTPs, software tokens) ensures only authorised users can establish VPN connections — username/password alone is insufficient.
Multiple exit nodesVPN traffic can exit from different servers worldwide, making it difficult to trace the origin of data and protecting user privacy.

Evaluate the Use of a VPN (3.1.5 — Obj 3)

✅ Advantages
+
Secure remote access: Staff can access the private network from anywhere as though physically on site. Data is encrypted in transit — prevents unauthorised interception.
+
Data confidentiality: Encryption ensures sensitive data (financial records, personal data) cannot be read if intercepted on a public network.
+
Cost-effective: Cheaper than dedicated private leased lines — uses existing public internet infrastructure with encryption on top.
+
Privacy & anonymity: Hides user's real IP address — useful where censorship or surveillance is a concern.
+
Bypass geographic restrictions: Can access content or services restricted to specific countries.
❌ Disadvantages
−
Reduced speed: Encryption/decryption overhead and routing through VPN server reduces connection speed, particularly noticeable for large files.
−
Setup cost and complexity: Requires VPN server, client software licences, and IT expertise to configure and maintain.
−
Single point of failure: If the VPN server fails, all remote users lose access to the private network.
−
Trust in VPN provider: VPN provider can theoretically log user activity — not all providers are trustworthy.
−
Incompatibility: Some services detect and block VPN traffic; not all legacy software supports VPN connections.

Social Implications of VPN / Remote Working (S/E, AIM 9)

VPNs enabling remote working have significant social implications — the IB expects you to discuss both positive and negative impacts:

✅ Positive Social Impacts

Improved work-life balance: Time saved on commuting can be spent with family, on hobbies, or other personal activities → improved wellbeing.
+

Key Definitions (3.1.6 — Obj 1)

PROTOCOL A set of rules and standards that govern how data is formatted, transmitted, received, and acknowledged between devices on a network. Protocols ensure that devices from different manufacturers can communicate effectively.
Examples: TCP/IP, HTTP, HTTPS, FTP, SMTP, DNS, POP3, IMAP
DATA PACKET A small unit of data that is transmitted over a network. A larger file or message is broken into many packets before transmission, and reassembled at the destination. Each packet contains a header, payload (data), and trailer.

Why Protocols are Necessary (3.1.7 — Obj 3)

Without protocols, devices from different manufacturers using different operating systems could not communicate. Protocols address several critical problems:

ProblemWhat it isHow protocols solve it
Data IntegrityData may become corrupted during transmission (bit errors due to noise, interference)Protocols include checksums/CRC in each packet — receiver verifies the checksum. If incorrect, the packet is rejected and re-requested.
Flow ControlA fast sender could overwhelm a slow receiver's buffer, causing data lossTCP uses sliding window flow control — the receiver advertises how much buffer space it has, and the sender won't exceed this.
DeadlockTwo nodes both waiting for the other to send first — neither proceeds, communication freezesProtocols define handshaking procedures (e.g. TCP 3-way handshake) to establish communication order and timeouts.
CongestionToo many packets on the same network segment simultaneously causes delays and packet lossRouters and protocols detect congestion and throttle transmission rates, re-route packets via less congested paths.
Error CheckingDetecting and correcting transmission errorsChecksums in headers/trailers detect errors. TCP requests retransmission of corrupted or missing packets. UDP does not (speed priority).

Why Transmission Speed Varies (3.1.8 — Obj 3)

▸
Network congestion: When many packets share the same route, queues form at routers → increased latency and reduced throughput
▸
Bandwidth: The maximum data transfer capacity of the connection. A low-bandwidth connection limits speed regardless of other factors.
▸
Distance: Signals weaken and take longer over greater distances. WANs are inherently slower than LANs for this reason.
▸
Number of hops (routers): Each router a packet passes through adds processing delay. More hops = more latency.
▸
Transmission medium: Fibre optic > copper cable > wireless in terms of speed and reliability.
▸
Protocol overhead: Protocols add headers/trailers to each packet. More complex protocols add more overhead, reducing effective data throughput.

Why Compression is Necessary (3.1.9 — Obj 3)

DATA COMPRESSION The process of reducing the size of a file or data stream by encoding it more efficiently, so that it contains fewer bits without (lossless) or with acceptable (lossy) loss of information.
▸
Reduces transmission time: Smaller files take less time to transmit over a network, especially important on slow connections or mobile networks.
▸
Reduces bandwidth usage: More data can be sent simultaneously over a fixed-bandwidth connection, reducing costs (especially for mobile data with caps).
▸
Enables richer media: Video streaming (e.g. Netflix) would be impractical without compression — H.264/HEVC compression makes HD video feasible over standard broadband.
🌍 S/E & INT: Compression has enabled information to be disseminated more rapidly across the globe. News, media, and knowledge that once took hours to transmit can now be shared in seconds. This accelerates globalization and democratises access to information — but also raises concerns about data quality and whether lossy compression distorts information.

How Packet Switching Works (3.1.11 — Obj 3)

Packet switching is the method by which data is broken into packets and transmitted independently across a network. This is in contrast to circuit switching (like traditional phone calls) where a dedicated path is reserved for the entire duration of a call.

1
The whole data (e.g. file, email, webpage) is split into equal-sized chunks called packets
2
Each packet is given a header containing: source IP address, destination IP address, packet sequence number, protocol type, checksum
3
Each packet is sent individually and independently — no dedicated circuit; packets compete for router resources
4
Routers examine each packet's header and forward it along the best available path (not necessarily the same path for every packet)
5
Packets may take different routes to reach the same destination — one packet might go via Tokyo, another via London
6
If a route becomes unavailable (e.g. router fails), individual packets can be automatically re-routed via alternative paths — the network is resilient
7
Packets may arrive out of order at the destination (because they took different routes of varying speeds)
8
The destination device uses the sequence numbers in the headers to re-order the packets into the correct sequence
9
The packets are reassembled into the complete original data
10
If any packets are missing or corrupt (detected by checksum), the destination requests those specific packets to be re-sent

Advantages of Packet Switching

✓
Efficient use of bandwidth: Multiple users' packets can share the same network links simultaneously
✓
Resilient: If one route fails, packets are automatically re-routed — no single point of failure
✓
Error recovery: Missing or corrupted packets can be individually re-requested and re-sent
✓
No reserved bandwidth: Unlike circuit switching, bandwidth is only used when data is actually being sent
✏️ IB Exam answer structure for "Explain how packet switching works" (4–6 marks): (1) Data split into packets. (2) Each packet has a header with source/dest IP, packet number, checksum. (3) Packets sent independently along best path. (4) May take different routes. (5) May arrive out of order. (6) Reassembled using sequence numbers. (7) Missing packets re-sent. — Each point with a reason/development earns marks.

Advantages and Disadvantages of Wireless Networks (3.1.12)

✅ Advantages
+
Mobility: Users can move freely within the network's coverage area while staying connected (smartphones, laptops on the move)
+
Easy to expand: Adding new devices does not require laying new cables — just connect to the existing Wi-Fi network
+
Cost of installation: Cheaper to install in existing buildings where cable runs would be expensive or disruptive
+
Changes in working patterns: Enables hot-desking, remote working, collaborative work from different locations (S/E)
+
New social activities: Social networking, location-based apps, instant communication anywhere
+

❌ Disadvantages
−
Security risks: Radio signals broadcast in all directions — anyone nearby can attempt to intercept data. Rogue access points and eavesdropping are real threats (S/E)
−
Interference: Other wireless devices (microwaves, neighbouring networks, Bluetooth) can degrade signal quality and speed
−
Speed and reliability: Generally slower and less reliable than wired connections; speed decreases with distance and obstructions
−
Limited range: Wi-Fi access points have limited range; large areas require multiple access points
−
Health concerns (S/E): Some people are concerned about long-term health effects of constant exposure to radio frequency (RF) radiation, though current research does not confirm harm at typical Wi-Fi levels

Hardware & Software Components of a Wireless Network (3.1.13)

Hardware Components:
H
Wireless router: Connects the wireless network to a wired network (e.g. the internet). Manages IP addressing and traffic routing. Acts as an access point in home networks.
H
Wireless Access Point (AP): Extends a wired network wirelessly. Used in larger networks where one router isn't enough — e.g. schools use multiple APs per building.
H
Wireless NIC (Network Interface Card): Hardware component inside each device (laptop, phone) that enables wireless communication. Has a unique MAC address.
H
Wireless bridge: Connects two separate wired network segments wirelessly — avoids the need to lay cables between buildings.
H
Wireless switches/hubs: Network switches that include wireless capability for managing and connecting wireless devices.
Software Components:
S
Network Operating System (NOS): Manages network resources, access control, and communication protocols. Vendors include VMware, Microsoft, DriveNets.
S
WPA/WPA2/WPA3 drivers: Software implementing Wi-Fi security protocols on each device. WPA (Wi-Fi Protected Access) = security standard for wireless devices.
S
Router firmware: Low-level software built into the router hardware — controls its basic functions, settings, and security features.
S
Configuration and management software: Web-based interface or dedicated software to configure SSID, security settings, access control, and DHCP settings.

1. Encryption (WEP → WPA → WPA2 → WPA3)

WPA2Wi-Fi Protected Access 2✅ Strong (currently standard)Uses AES encryption. Strong and widely supported. Current minimum standard for all Wi-Fi networks.
WPA3Wi-Fi Protected Access 3✅✅ Very strongLatest standard. Stronger AES-192/256, protects against brute-force and KRACK attacks. Forward secrecy.
✅ Encryption — Advantages
+
Even if data is intercepted by an attacker, it is unreadable without the decryption key
+
Protects sensitive data (passwords, financial details, personal information) in transit
+
Modern WPA2/WPA3 is computationally infeasible to brute-force with current technology
+
Widely supported across all modern devices — transparent to users
❌ Encryption — Disadvantages
−
Encryption/decryption adds processing overhead, slightly reducing network speed
−
Older encryption standards (WEP, WPA) are easily cracked by modern tools — a false sense of security if not upgraded
−
Encryption only protects data in transit — doesn't protect against attacks once data is on the destination device
−
Key management can be complex; if the key is compromised, all security is lost

2. UserID and Password Authentication

Users must provide a unique identifier (UserID/username) and secret password before being granted access to the network. This is the most basic and widely used authentication method.

✅ Advantages
+
Simple to implement — built into virtually all operating systems and network hardware
+
Prevents casual or accidental access by unauthorised users
+
Can be combined with other methods (two-factor authentication = UserID + password + OTP)
+
Individual accounts can be created, modified, or revoked instantly — easy access management
❌ Disadvantages
−
Weak passwords can be guessed, brute-forced, or obtained via phishing — entirely reliant on the user choosing and protecting a strong password
−
Passwords can be shared between users, undermining individual accountability
−
No control over which device connects — an authorised user could log in from an insecure device
−
Credentials stored on a server are a high-value target; if the server is breached, all user accounts are at risk

3. Trusted MAC Address Filtering

MAC ADDRESS A Media Access Control (MAC) address is a unique hardware identifier permanently assigned to a network interface card (NIC) at manufacture. It is a 48-bit address written as 6 pairs of hexadecimal digits, e.g. A4:C3:F0:85:AC:2D. No two NICs should have the same MAC address.

MAC address filtering (also called a MAC whitelist) works by configuring the wireless router/access point to only accept connection requests from devices whose MAC addresses appear on an approved list. Any device not on the list is denied access.

1
Network administrator compiles a list of MAC addresses of all authorised devices
2
This whitelist is entered into the router/AP configuration
3
When a device attempts to connect, the router checks its MAC address against the whitelist
4
If the MAC address matches — access granted. If not — access denied, even with the correct password
✅ Advantages
+
Provides hardware-level access control — even if someone knows the Wi-Fi password, their device won't connect if the MAC isn't whitelisted
+
Acts as an additional layer of security (defence in depth) when combined with encryption
+
Provides an audit trail — the network administrator knows exactly which devices are permitted
+
Stops casual intruders who may know the Wi-Fi password from connecting on unknown devices
❌ Disadvantages
−
MAC spoofing: An attacker can use software to copy ("spoof") a trusted device's MAC address, making their device appear to be an authorised one — bypassing the filter entirely
−
Scalability: Managing a whitelist for large organisations (hundreds of devices) is time-consuming and error-prone
−
Not practical for guest access: Every visitor's device must be manually added and removed — creates administrative burden
−
MAC addresses are transmitted in plain text in wireless frames — easy for an attacker with network monitoring tools to see them and identify which ones are trusted
✏️ IB Exam — "Evaluate trusted MAC address as one method of network security" (4 marks): Start with how it works: "The router maintains a whitelist of approved MAC addresses; only devices with listed MAC addresses can connect [1 mark]. This is effective because [advantage + reason, 1 mark]. However, a limitation is MAC spoofing — an attacker can use software to copy a trusted device's MAC address [1 mark], making the filter ineffective against a determined attacker [development, 1 mark]."
